Phase 10 — Clan War MVP

목표:

Clan끼리 토크배틀 기반으로 경쟁할 수 있는 가문전 MVP를 구현합니다.

이번 단계는 실시간 대규모 전쟁이 아니라, 기존 토크배틀 시스템과 연결 가능한 비동기 가문전 구조를 만듭니다.

---

핵심 원칙

* 기존 토크배틀 로직을 크게 수정하지 말 것
* 기존 Persona XP 수정 금지
* 기존 Clan XP 수정 금지
* 기존 Clan Ranking 수정 금지
* 기존 Clan Memory/Wisdom 수정 금지
* 이번 단계는 Clan War 생성/참가/결과 기록까지만 구현
* 실시간 매칭/시즌/토너먼트는 아직 구현하지 않음

---

1. Clan War 개념

가문전은 두 가문이 하나의 주제로 토론 점수를 겨루는 경쟁입니다.

기본 흐름:

1. 가문장이 가문전 생성
2. 상대 가문 선택 또는 공개 도전 생성
3. 각 가문원이 참여
4. 기존 토크배틀 결과 또는 가문전 전용 발언 점수를 합산
5. AI 심판 결과를 기반으로 승패 결정
6. 결과 기록

3. 권한 규칙

가문전 생성:

* owner 또는 elder만 가능

공개 도전 참가:

* 상대 가문의 owner 또는 elder만 수락 가능

가문전 참여:

* 해당 가문 멤버만 가능

결과 확정:

* MVP에서는 조건 충족 시 서버가 자동 확정
* 관리자 수동 확정은 아직 구현하지 않음

---

4. API

POST /clan-wars

* 가문전 생성

GET /clan-wars

* 공개/진행/완료 가문전 목록

GET /clan-wars/:id

* 상세 조회

POST /clan-wars/:id/accept

* 공개 도전 수락

POST /clan-wars/:id/join

* 가문원이 가문전에 참여

POST /clan-wars/:id/submit

* 참가자가 주장/발언 제출

POST /clan-wars/:id/complete

* 조건 충족 시 결과 계산

5. MVP 점수 방식

처음에는 복잡한 실시간 토론 대신 단순 제출형으로 구현합니다.

참가자는 가문전 주제에 대해 짧은 주장을 제출합니다.

AI 심판은 각 제출을 평가합니다.

평가 항목:

* 논리성
* 설득력
* 근거
* 공감
* 표현력

각 항목 0~10점

참가자 score = 합산 점수

가문 score = 참가자 score 총합 또는 상위 3명 평균

MVP 추천:

* 상위 3명 평균

이유:
멤버 수 많은 가문이 무조건 유리해지는 문제 방지

---

6. AI 심판

AI 호출은 제출마다 하지 말고,
complete 시점에 한 번에 평가합니다.

입력:

* topic
* challenger submissions
* opponent submissions

출력:
{
"challenger_score": 82,
"opponent_score": 76,
"winner": "challenger",
"judge_summary": "...",
"challenger_feedback": "...",
"opponent_feedback": "..."
}

안전:

* 원문을 영구 공개하지 말 것
* 결과 요약만 저장
* 민감정보/개인정보가 포함되면 저장 전 필터링

8. 보상은 아직 최소화

이번 단계에서는 Clan XP 보상은 작게만 적용하거나 아예 제외해도 됨.

추천:

* 승리 가문 +100 Clan EXP
* 패배 가문 +30 Clan EXP
* 참가자 기여도 +10

단, 기존 Clan XP 로직과 충돌하지 않도록 별도 source_key로 처리

source_key 예:
clan_war:{warId}:result:{clanId}
clan_war:{warId}:participant:{userId}
